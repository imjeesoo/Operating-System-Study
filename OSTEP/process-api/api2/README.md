## 2. api2.c : 파일 디스크립터 상속 및 파일 오프셋 공유 확인

### 1) 실험 목적  

open() 시스템콜로 파일을 연 뒤 fork()를 호출했을때, 부모 프로세스와 자식프로세스가 동일한 파일 디스크립터를 사용할 수 있는지,
그리고 동시에 파일에 write 작업을 수행할 수 있는지 확인한다.
또한 fork() 이후 부모와 자식 프로세스가 파일 오프셋을 공유하는지 확인한다.

### 2) 구현 세부 사항 (api2.c)  

open() 시스템콜을 이용하여 test.txt파일을 생성하고 연다.  
파일을 연 뒤 fork() 호출하여 자식프로세스를 생성한다. 
자식프로세스와 부모프로세스는 각각 동일한 파일디스크립터(fd)를 사용하여 파일에 문자열을 작성한다.  
wait()을 사용하지 않았기 때문에 부모와 자식 프로세스의 실행 순서는 보장되지 않는다.  

### 3) 결과 분석 및 결론  
<img width="575" height="160" alt="image" src="https://github.com/user-attachments/assets/94563062-691a-4d3e-b066-169624a1446d" />

fork() 이후 부모와 자식 프로세스 모두 동일한 파일디스크립터(fd)에 접근 가능하다.  
그리고 모두 파일에 쓰기 작업을 수행할 수 있다.  
하지만 실행순서는 보장되지 않는다. 실행시 마다 문자열의 순서가 달라질 수 있다.  
파일쓰기 작업이 덮어써지지 않고 작성이 되는것은 fork()이후 부모와 자식프로세스가 동일한 open file description을 공유하기 
때문에 file offset 또한 공유되어 한 프로세스가 write를 수행한 다음 위치부터 write를 수행한다.  
이 실험은 운영 체제(UNIX와 같은)에서 파일 디스크립터 상속이 어떻게 작동하는지,  
그리고 부모 프로세스와 자식 프로세스가 fork() 후 동일한 open file description을 공유한다는 것을 보여준다.
